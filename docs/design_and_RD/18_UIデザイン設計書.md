# **Training Bot RAG Hub UI/UXデザイン設計書 v1.0**

## **1. デザインコンセプト**

### **1.1 採用コンセプト**

Institutional AI Terminal

Bloomberg Terminal、Palantir Foundry、OpenAI Playgroundを参考にした、金融AI分析基盤向けのUIとする。

本UIは「売買する画面」ではなく、「AIが提示した根拠・リスク・引用・監査情報を確認する画面」として設計する。

---

## **2. デザイン原則**

### **2.1 最重要原則**

- RAG回答よりも根拠を目立たせる
- BUY / SELL を強調しすぎない
- Risk Levelを常に可視化する
- Citationを必ず見える位置に置く
- order_permission=false をUI上でも明示する
- 高リスク時は視覚的に警告する
- 監査ログに辿れる導線を常設する

## **7. 主要画面構成**

## **7.1 Dashboard**

目的：RAG Hubの稼働状況、リスク状況、Provider使用状況を俯瞰する。

主要要素：

- Query Count
- Avg Latency
- Monthly Cost
- High Risk Count
- Provider Usage
- Recent RAG Queries
- Guardrail Blocks
- Audit Alert

デザイン方針：

重要KPIはMetric Cardで表示。  
Risk Alertは右上に固定表示する。

---

## **7.2 AI分析画面**

最重要画面。

### **レイアウト**

```text
┌────────────┬────────────────────┬──────────────┐
│ Query Panel│ AI Analysis Panel  │ Citation Panel│
│ Filter     │ Summary            │ Sources       │
│ History    │ Risk               │ Reliability   │
│ Provider   │ Factors            │ Recency       │
└────────────┴────────────────────┴──────────────┘
```

### **中央パネル表示順**

1. Warning Banner
2. AI Summary
3. Risk Level
4. Confidence Meter
5. Supporting Factors
6. Opposing Factors
7. Similar Cases
8. Guardrail Status

### **UI方針**

BUY / SELLを目立たせすぎない。  
支持材料と反対材料を同じ視覚階層で表示する。  
AI SummaryよりRiskとCitationを近い位置に置く。

---

## **7.3 Bot検証画面**

目的：Training Botの仮シグナルに対して、RAGが根拠・反対材料・リスクを提示する。

主要要素：

- Bot Signal
- Strategy ID
- Features
- RAG Explanation
- Supporting Factors
- Opposing Factors
- Similar Cases
- order_permission=false
- Human Review Status

デザイン方針：

Bot Signalは大きく表示するが、発注ボタンは置かない。  
order_permission=falseを常時表示する。

---

## **7.4 Similar Cases画面**

目的：現在相場に近い過去ケースを比較する。

主要要素：

- Similarity Score
- Period
- Market Condition
- After Move
- Max Drawdown
- Risk Notes
- Citation

デザイン方針：

Similarity ScoreはProgress Barで表示。  
Max Drawdownは必ず赤系で表示する。  
利益よりも損失方向の情報を目立たせる。

---

## **7.5 RAG履歴画面**

目的：過去の問い合わせ、回答、引用、Provider、Guardrail結果を追跡する。

主要カラム：

- Date
- Query
- Symbol
- Risk Level
- Confidence
- Provider
- Cost
- Latency
- Guardrail Status

デザイン方針：

監査画面として検索性を優先する。  
テーブル密度は高め。  
フィルターは左固定。

---

## **7.6 回答詳細画面**

目的：1つのRAG回答の全情報を監査可能にする。

表示項目：

- Query
- Response
- Supporting Factors
- Opposing Factors
- Risk Level
- Confidence
- Citations
- Provider Usage
- Token Cost
- Latency
- Guardrail Result
- trace_id

---

## **7.7 Provider評価画面**

目的：OpenAI、Claude、Gemini、Mistralなどの品質・コスト・安全性を比較する。

主要指標：

- Schema Success Rate
- Citation Accuracy
- Hallucination Rate
- Risk Coverage
- Latency
- Cost per Query
- Safety Violation Rate

デザイン方針：

単なるランキングではなく、用途別の最適Providerを見せる。  
リスクレビュー、通常要約、外部情報要約で比較軸を分ける。

---

## **8. コンポーネント設計**

### **8.1 RiskBadge**

用途：Risk Level表示

仕様：

- LOW: Green
- MEDIUM: Blue
- HIGH: Amber
- CRITICAL: Red
- 角丸: 999px
- 高さ: 24px
- padding: 8px 12px

---

### **8.2 ConfidenceMeter**

用途：信頼度表示

仕様：

- 0.0〜1.0
- Progress Bar + 数値
- 0.5未満はWarning表示
- Tooltipで根拠数、ソース品質、鮮度を表示

---

### **8.3 CitationCard**

用途：参照ソース表示

表示項目：

- Source Type
- Source Name
- Reliability Score
- Recency Score
- Used Reason
- Link / Detail Button

---

### **8.4 FactorCard**

用途：支持材料・反対材料表示

種類：

- Supporting Factor
- Opposing Factor
- Neutral Factor

方針：

支持材料だけを強くしない。  
反対材料も同等の視認性にする。

---

### **8.5 GuardrailStatus**

用途：安全制御結果表示

状態：

- PASS
- WARNING
- BLOCKED

BLOCKEDは赤。  
WARNINGはAmber。  
PASSはGreen。

---

### **8.6 ProviderBadge**

用途：使用Provider表示

例：

- OpenAI
- Claude
- Gemini
- Mistral
- Local LLM

Provider名、Model名、Fallback有無を表示する。

### **9.2 Secondary Button**

用途：

- Sell Now

- Auto Trade

## **10. Warning設計**

### **10.1 常時表示Warning**

```text
この出力は投資助言ではなく、Training Bot検証用の参考情報です。
RAGは注文権限を持ちません。
```

### **10.2 高リスク時Warning**

```text
High Risk: 参照情報に不確実性または急変動リスクが含まれます。
根拠と反対材料を確認してください。
```

### **10.3 Citation不足時Warning**

```text
Citation不足: 十分な根拠ソースが取得できていません。
この回答のConfidenceは低く扱ってください。
```

## **16. 最終デザイン方針**

Training Bot RAG HubのUIは、チャート中心でもチャット中心でもなく、根拠中心のAI分析ターミナルとして設計する。

見た目は金融システムとして信頼できるDark Terminal UI。  
操作体験はAI Research Workspaceのように自然にする。  
そして全画面で、Risk、Citation、Confidence、Guardrailを見失わない設計にする。
