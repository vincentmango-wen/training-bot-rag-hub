# **5. データセット詳細**

## **RAG-EVAL-001 市場要約評価**

### **目的**

市場状況の要約能力を評価する。

### **入力例**

```json
{
  "symbol": "BTCUSDT",
  "market_data": {
    "rsi": 72,
    "volume_spike": true,
    "price_change_24h": 8.4
  },
  "retrieved_context": [
    "ETF流入増加",
    "出来高急増"
  ]
}
```

### **評価項目**

|**項目**|**配点**|
|---|---|
|要約精度|40|
|Citation整合性|20|
|金融妥当性|20|
|リスク記載|20|

合計100点

---

## **RAG-EVAL-002 Bot判断理由評価**

### **目的**

Botシグナル説明能力を評価する。

### **入力例**

```json
{
  "bot_signal": "BUY",
  "features": {
    "rsi": 28,
    "macd": "golden_cross",
    "volume_spike": true
  }
}
```

### **評価基準**

必須項目

- supporting_factors
- opposing_factors
- risk_level
- confidence

不足時は減点

---

## **RAG-EVAL-003 反対材料抽出評価**

### **目的**

Confirmation Biasを抑制できるか評価する。

### **入力**

```json
{
  "signal": "BUY"
}
```

### **期待例**

```json
{
  "opposing_factors": [
    "上位足下降トレンド",
    "出来高減少"
  ]
}
```

### **合格基準**

反対材料1件以上

# **最終方針**

評価データセットは単なるLLMベンチマークではなく、

```text
1. 金融妥当性
2. Citation整合性
3. Guardrail安全性
4. Schema安定性
5. Provider比較
6. コスト最適化
```

を継続的に測定する品質管理基盤として運用する。

  

Training Bot RAG Hubでは「高スコア」よりも、

```text
誤発注誘発ゼロ
Secret漏洩ゼロ
order_permission=true ゼロ
Prompt Injection突破ゼロ
```

を最優先品質指標とする。これは企画書・要件定義書・テスト計画書で定義された最重要原則と一致する。
