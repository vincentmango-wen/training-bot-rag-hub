# **Training Bot RAG Hub 詳細ワイヤーフレーム・コンポーネント実装仕様書 v1.0**

## **1. 前提**

本UIは、Training Bot RAG Hubを「売買画面」ではなく「AI分析・根拠確認・監査画面」として設計する。

重要表示項目は以下。

- Risk Level
- Confidence
- Citation
- Supporting Factors
- Opposing Factors
- Similar Cases
- Provider Usage
- Guardrail Status
- order_permission=false

RAGは注文実行せず、判断材料のみを提供する。これは要件定義上の最重要原則である。

# **4.1 RiskBadge**

## **目的**

Risk Levelを即時認識させる。

## **Props**

```ts
type RiskLevel = "LOW" | "MEDIUM" | "HIGH" | "CRITICAL";

type RiskBadgeProps = {
  level: RiskLevel;
  size?: "sm" | "md" | "lg";
};
```

## **表示仕様**

|**level**|**style**|
|---|---|
|LOW|green|
|MEDIUM|blue|
|HIGH|amber|
|CRITICAL|red|

## **実装例**

```tsx
export function RiskBadge({ level, size = "md" }: RiskBadgeProps) {
  const styles = {
    LOW: "bg-green-950 text-green-400 border-green-800",
    MEDIUM: "bg-blue-950 text-blue-400 border-blue-800",
    HIGH: "bg-amber-950 text-amber-400 border-amber-800",
    CRITICAL: "bg-red-950 text-red-400 border-red-800",
  };

  return (
    <span className={`inline-flex items-center rounded-full border px-3 py-1 text-xs font-medium ${styles[level]}`}>
      {level}
    </span>
  );
}
```

# **4.2 ConfidenceMeter**

## **Props**

```ts
type ConfidenceMeterProps = {
  value: number;
  showLabel?: boolean;
};
```

## **仕様**

- valueは0〜1
- 0.5未満はWarning
- 0.8以上はHigh Confidence
- 色だけでなくラベルも表示

## **実装例**

```tsx
export function ConfidenceMeter({ value, showLabel = true }: ConfidenceMeterProps) {
  const percent = Math.round(value * 100);

  const label =
    value < 0.3 ? "Low" :
    value < 0.6 ? "Medium" :
    value < 0.8 ? "High" : "Very High";

  return (
    <div className="space-y-1">
      {showLabel && (
        <div className="flex justify-between text-xs text-slate-400">
          <span>Confidence</span>
          <span>{label} / {percent}%</span>
        </div>
      )}
      <div className="h-2 rounded-full bg-slate-800">
        <div
          className="h-2 rounded-full bg-blue-500"
          style={{ width: `${percent}%` }}
        />
      </div>
    </div>
  );
}
```

# **4.5 GuardrailStatus**

## **Props**

```ts
type GuardrailState = "PASS" | "WARNING" | "BLOCKED";

type GuardrailStatusProps = {
  status: GuardrailState;
  orderPermission: false;
  reason?: string;
};
```

## **仕様**

orderPermissionは常にfalse。  
trueが渡された場合は例外を投げるか、CRITICAL表示にする。

## **実装例**

```tsx
export function GuardrailStatus({
  status,
  orderPermission,
  reason,
}: GuardrailStatusProps) {
  if (orderPermission !== false) {
    throw new Error("RAG UI violation: orderPermission must be false");
  }

  const styles = {
    PASS: "border-green-800 bg-green-950 text-green-400",
    WARNING: "border-amber-800 bg-amber-950 text-amber-400",
    BLOCKED: "border-red-800 bg-red-950 text-red-400",
  };

  return (
    <div className={`rounded-xl border p-4 ${styles[status]}`}>
      <div className="font-semibold">Guardrail: {status}</div>
      <div className="mt-1 text-xs">order_permission=false</div>
      {reason && <div className="mt-2 text-xs opacity-80">{reason}</div>}
    </div>
  );
}
```

# **7. 実装ルール**

## **7.1 禁止**

- RAG結果から直接注文ボタンを表示しない
- BUY / SELLを単独で大きく表示しない
- Supporting Factorsだけを強調しない
- Citationなし回答を通常表示しない
- order_permission=trueを許容しない

## **7.2 必須**

- Risk Levelを全回答に表示
- Confidenceを全回答に表示
- Citationを全回答に表示
- Guardrail Statusを表示
- Provider / Model / Cost / Latencyを詳細画面で表示
- trace_idをコピー可能にする

# **9. 受入条件**

- Risk Levelが全主要画面に表示される
- Confidenceが全主要画面に表示される
- Citationが確認できる
- Guardrail Statusが確認できる
- order_permission=falseがBot検証画面と回答詳細画面で確認できる
- RAG結果から注文操作ができない
- Provider Usageを詳細画面で確認できる
- 履歴から回答詳細へ遷移できる
- 画面上で投資助言と誤解される表現がない
