# 2. 見積もり結論

## 2.1 結論

Training Bot RAG HubのMVPは、ローカル実行を前提にすれば、**月額 $5〜$50 程度**で開始可能である。

ただし、以下を増やすとコストは急増する。

- RAG問い合わせ回数
    
- 1回あたりの入力Context量
    
- 高性能LLMの利用比率
    
- Provider比較評価の頻度
    
- 外部ニュースAPI / SNS APIの利用
    
- クラウド常時稼働
    
- 大量バックテスト時のLLM呼び出し
    

最初のMVPでは、OpenAI APIを第一候補にし、以下の構成を推奨する。

```text
Local Docker Compose
+ PostgreSQL / pgvector
+ Redis
+ OpenAI text-embedding-3-small
+ OpenAI GPT-5.4 mini
+ 高リスク時のみGPT-5.4またはClaude Sonnet
```

---

## 2.2 推奨予算

|フェーズ|月額目安|内容|
|---|--:|---|
|Phase 1: ローカルMVP|$5〜$30|内部RAG、Botログ、戦略文書、少量Query|
|Phase 2: 実用検証|$30〜$100|類似ケース検索、Bot説明生成、Provider比較|
|Phase 3: 外部情報RAG|$100〜$300|ニュース、予測市場、外部データ取込|

# 13. コスト削減策

## 13.1 LLM費削減

|施策|効果|
|---|---|
|GPT-5.4 miniを標準化|通常Query費を抑える|
|高リスク時のみ上位モデル|品質と費用を両立|
|Context圧縮|Input token削減|

|回答Cache|同一Queryの再生成防止|
|Batch API利用|評価・一括処理費を削減|
|Provider比較頻度制限|無駄な評価費を抑える|

|施策|効果|

|差分Indexing|再Embedding防止|
|重複除去|Token削減|
|Chunkサイズ最適化|検索品質と費用を両立|
|largeは必要時のみ|高コスト化を防ぐ|

# 17. 最終判断

Training Bot RAG HubのMVPは、コスト面では十分現実的である。

理由：

- Embedding費は非常に安い
    
- ローカル実行ならインフラ費はほぼゼロ
    
- OpenAI GPT-5.4 mini中心なら月額数十ドルで運用可能
    
- Provider比較も小規模なら数ドル〜十数ドルで実施可能
    
- 高性能モデルを限定利用すれば品質と費用を両立できる
    

ただし、以下は必ず守る。

```text
1. RAG Queryの月額上限を設定する
2. 通常Queryはminiモデルを使う
3. 高リスク時のみ上位モデルを使う
4. Embeddingはsmallを標準にする
5. Provider比較は月1回までにする
6. 外部データAPIはMVPでは無料ソース中心にする
7. クラウド常時稼働は詳細設計後に判断する
```

結論：

```text
MVP予算は月額 $50以内を目標にする。
実用検証フェーズでも月額 $100以内を第一目標にする。
Active Backtestや外部情報RAGを増やす場合のみ、月額 $300程度まで許容する。
```
